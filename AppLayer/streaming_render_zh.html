01 / 核心操作机制

LiteRT-LM 的异步特性决定了它不能直接“返回值”，而是通过事件流进行通信。

  • 线程分发 (Marshalling)： 内核在产出 Token 后，会自动通过虚幻的 AsyncTask(ENamedThreads::GameThread) 将字符送回主线程。开发者可以直接在回调中更新控件。
  • 首字延迟对冲： 借助于流式特性，即便模型需要 2 秒才能算完 100 个字，用户在 100ms 内就能看到第一个字跳动。
02 / 源码实战 Demo
// 参考 WinyunqDialogueWidget.cpp
void UWinyunqDialogueWidget::ExecuteChat(FString UserMsg) {
    // 1. 定义流式消费 Lambda
    FLiteRtLmChunkCallback OnChunk = ...Lambda([this](const FString& Chunk) {
        HistoryBuffer += Chunk;
        Text_Display->SetText(FText::FromString(HistoryBuffer));
    });

    // 2. 发送异步指令
    CurrentNPC->SendChatMessage(UserMsg, OnChunk);
}
