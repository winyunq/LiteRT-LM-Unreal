强力约束解码 (Constrained Decoding) 与业务触发

Physical Atlas / 指令生成与转换图解
01 / 概念深度

为了防止 AI 在关键指令处输出废话（Hallucination），插件支持在 Token 生成的概率阶段进行硬约束。这确保了产出的每一位字符都符合预设的语法。

  • JSON 模式： 自动补全括号，强制输出合法的 JSON 键值对。支撑了业务层的工具调用逻辑。
  • 工具注入： 插件通过 json_preface 技术向 AI 宣告虚幻内部可用的函数清单，实现“思考即行动”。
02 / 源码实战 Demo

参考业务层如何定义指令约束并消费结果：

// 1. 设置强约束类型
FLiteRtLmSamplingParams Params;
Params.ConstraintType = ELiteRtLmConstraintType::Json;

// 2. 注入可用工具定义 (UmgMcp 标准)
FString ToolsJson = "[{\"name\":\"attack\", \"parameters\":{...}}]";

// 3. 在 OnDone 中消费结构化指令
void OnInferenceDone(const FLiteRtLmResult& Result) {
    for (auto Call : Result.ToolCalls) {
        FString FuncName = Call->GetStringField("name");
        ExecuteInternalAction(FuncName, Call->GetObjectField("parameters"));
    }
}
