Operation 03 / Session Management

会话与 KV CACHE

持久化记忆的分配与物理销毁规程

Sequence / 会话操作时序
            sequenceDiagram
                participant NPC as Actor (Level 3)
                participant SUB as Subsystem (Level 2)
                participant K as Kernel (Level 1)
                
                NPC->>SUB: 1. GetOrCreateSession(this)
                SUB->>K: 2. 申请显存 Slot
                K-->>SUB: 3. 返回 void* 物理句柄
                SUB->>SUB: 4. 建立 Pointer -> Handle 映射
                Note over NPC, K: 会话已就绪，开始推理循环
                NPC->>NPC: 执行多次对话 (Send/Render)
                
                NPC->>SUB: 5. 关卡退出/对象销毁 (EndPlay)
                SUB->>K: 6. ReleaseSession(this)
                K->>K: 7. 物理清空 KV Cache 显存
01 / 具体操作动作
动作 A: 指针绑定

调用 GetOrCreateSession。插件会自动识别当前 Actor 的内存地址。只要地址不变，AI 就拥有“连续记忆”。

动作 B: 物理回收

必须在 EndPlay 或对象析构时调用 ReleaseSession。由于显存是物理分配的，虚幻的 GC 无法自动回收它。

02 / 源码实战 Demo
// 1. 业务开始：建立会话
void* SessionKey = Subsystem->GetOrCreateSession(this);

// 2. 业务结束：强制归还显存 (重要操作)
void ANPCCharacter::EndPlay(...) {
    Super::EndPlay(Reason);
    if (ULiteRtLmSubsystem* SUB = ...) {
        SUB->ReleaseSession(this);
    }
}
