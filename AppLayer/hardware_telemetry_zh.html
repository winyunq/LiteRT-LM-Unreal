Operation 05 / Hardware Defense

硬件遥测防御

DXGI 实时轮询与系统熔断规程

Sequence / 硬件遥测轮询时序
            sequenceDiagram
                participant HUD as UI/HUD (Level 3)
                participant SUB as Subsystem (Level 2)
                participant DX as DXGI Driver (Kernel)
                
                loop 每隔 1.0 秒
                    HUD->>SUB: 1. QueryAvailableVramMB()
                    SUB->>DX: 2. 穿透查询物理 Budget
                    DX-->>SUB: 3. 返回真实物理剩余
                    SUB-->>HUD: 4. 返回 int32 (MB)
                    alt 显存安全
                        HUD->>HUD: 显示绿色指示灯
                    else 显存告急
                        HUD->>HUD: 触发业务降级/弹窗警告
                    end
                end
01 / 具体操作动作
动作 A: 实时采样

在 HUD 的 NativeTick 中调用。插件内部通过 **DXGI** 确保该调用延迟极低（< 0.1ms），不会阻塞渲染管线。

动作 B: 物理熔断

当显存水位低于 1GB 时，业务层应立即停止向插件发送 LoadModel 指令，转而使用 cpu 后端。

// 业务层标准遥测写法 (DemoTavernHUD.cpp)

    // 静态获取物理指标

    // 同步更新显示逻辑
    StatusText = FString::Printf(TEXT("VRAM: %d MB"), Vram);
}
