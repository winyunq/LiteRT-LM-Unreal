上下文窗口管理

128K 物理标准与滑动窗口策略

Physical Atlas / 滑动窗口物理机制
01 / 概念深度

上下文窗口决定了 AI NPC 的“工作记忆”长度。在 LiteRTDemo 中，我们采用了 128K (131,072) Tokens 的物理标准。

  • 人格锚定： 插件内核会自动保护 System Prompt，即使发生滑动窗口截断，也不会丢失 NPC 的核心设定权重。
  • 滑动物理： 当显存槽位存满时，内核会物理驱逐最早的非系统消息。由于 128K 窗口极大，这在常规游戏流程中极难触发。
02 / 源码实战 Demo

参考 UmgMcpLiteRtLmAiProvider.cpp 如何定义物理界限：

// 1. 设置 128K 超长上下文标准
FLiteRtLmConfig Config = FLiteRtLmUnrealApi::GetAutoConfig();
Config.MaxNumTokens = 128 * 1024;  // 物理分配 ~1.5GB KV Cache

// 2. 限制单次回复长度
FLiteRtLmSamplingParams Params;
Params.MaxTokens = 4096; // 防止单次生成击穿预算

// 3. 业务层推送
FLiteRtLmUnrealApi::SendChatRequest(..., Params);
