LiteRtLm_CreateEngine 核心入口 

DLL_EXPORT void* LiteRtLm_CreateEngine(LiteRtLm_Config config);
  • 物理锁： 该函数执行时会立即锁定显存。若显存不足，返回 NULL。
  • ABI 防火墙： 返回 void* 句柄，禁止在 UE5 侧进行类型转换。
  • 调用时机： 建议在关卡加载或 Subsystem 初始化时执行，禁止在主循环中重复调用。

结构体 LiteRtLm_Config

max_num_tokens

决定 KV Cache 物理预分配大小。一旦分配，显存占用固定。

公式：显存 ≈ 权重 + (Tokens * 128 KB)

bOptimizeShader

UE5 必备项。设为 1 则加载时编译 PSO，消除推理首字掉帧。

注意: model_path 采用 mmap 映射。模型文件损坏或路径非法会导致进程直接崩溃，调用前必须验证文件物理存在。

LiteRtLm_GetAvailableBackends

环境探测接口。返回系统加速能力字符串（如 "gpu,cpu"）。

策略: If (!str.contains("gpu")) { config.num_threads = 16; }
